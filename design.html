Design

Introduction

When designing the website, I knew I wanted a cheerful and welcoming site to not intimidate users, and so chose casual page titles and headers. The ultimate goal in mind was to make a site that encouraged users to join or to express their ideas.

Site Map

Design reasoning

We defined medial rule based styles. The default css style is for small/mobile device. If the screen size is over 768px, then narrow/medium screen css will be applied, if the screen size is over 992px, then the normal/large screen style sheet will be applied. The breakpoints are choosen based on the mostly popular device sizes on the market also based on the page content(pictures are 300px wide).

For pages with pictures, grid layouts is being used to set number of columns per row. On mobile device, only one picture each row, on narrow device, we show 2 pictures per row, on normal device, we show 4 pictures per row.

On the contact us page, we use grid layout to make field label and input field side by side on normal and narrow screen. For mobile device, we defines different grid layout so label and field are vertically arranged.

On purpose page, the pictures and questions are displayed horizontally for narrow and normal screen. For mobile device, the picture will be displayed on top of question.

On design page, the sitemap picture width is set to 600px on narrow and normal screen and 300px on mobile device.

Menu System

To produce a responsive menu for different size of devices, we use inline-block to display them horizontally on screen when the size of the screen is big enough. Inline-block also automatically apply breakpoints as to the screen size, when the browser window resizes, the links will automatically break into multiple rows to fit the screen.

Design mock-ups

For each page, we have design mock-up for normal screen, narrow screen and mobile device. For pages with pictures, the number of pictures on each row is based on the picture size as well breakpoint width.
