Design

Introduction

When designing the website, I knew I wanted a cheerful and welcoming site to not intimidate users, and so chose casual page titles and headers. The ultimate goal in mind was to make a site that encouraged users to join or to express their ideas.

Site Map

Design reasoning

I chose three breakpoints: normal, narrow, and mobile. The default style is for the mobile breakpoint. If the screen width is over 768px, then the narrow style will be applied, and if the screen width is over 992px, then the normal style will be applied. The breakpoints were choosen based on popular device sizes on the market, and were also based on the pages' content (the pictures are 300px wide).

For pages with pictures, grid layouts were used to set the number of columns per row. On mobile devices, there is only one picture per row, for the narrow style, there are two pictures per row, and for the normal style, there are four pictures per row.

On the contact page, I used a grid layout to make the field label and the text box side by side on normal and narrow screens. For mobile devices, I defined a different grid layout so the label and text box are vertically arranged.

On the purpose page, the pictures and questions are displayed horizontally for the narrow and normal widths. For mobile devices, the pictures are displayed on top of the questions.

On the design page, the sitemap picture's width is set to 600px for narrow and normal widths and 300px on mobile devices.

All of these changes were to ensure all the non-text elements could fit on the screen without the need for a scrollbar, for convenience.

Menu System

To produce a responsive menu for different size of devices, I used inline-block to display them horizontally on screen when the screen is wide enough. Inline-block also automatically applies breakpoints based on screen size -- when the window resizes, the links automatically break into multiple rows to fit the screen.

Design mock-ups

I created design mock-ups for normal, narrow and mobile breakpoints. For pages with pictures, the number of pictures on each row was based on the pictures' sizes as well the breakpoint widths. I created unique designs for all the pages to be linked on the headers, and intended for the layouts of the footer pages to be the same. Some choices, such as the white text for page links, were scrapped for accessibility.
