Testing

Optimisation

The potential cause of performance issue is the number of images on each page. On design page, there are 18 mock-up pictures, if we display all these picures on the design page, the page loading will take longer. Instead of displaying pictures, we display hyperlinks and that will make page loading lot faster.

The pictures displayed on other pages have also been optimised with smaller file size and reasonable quality.

HTML Validation

Websites have been validated using online validation tools. Errors are caused by a missing html start tag and inappropriate use for label for attribute. After correction, the website is validated again and it passes the test.



Browser Testing

To test media rule based style, tests have been made using simulator of Chrome and the following are the screenshots of test results. The website works as expected under different screen sizes

Accessibility

Accessibility check has been carried out using WAVE. The test results showed 9 low contrast errors in header and footer. The cause of the errors is the menu font is not dark enough.

To find a suitable font color for accessibility, contrast checker is used and black color is a suitable font color.

After updating the menu font color to black, another accessibility test is done, this time it passes the test.
