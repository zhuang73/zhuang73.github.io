Testing

Optimisation

A potential cause of performance issues is the number of images on each page. On the design page, there are 18 design pictures. The screenshot below is the performance measurement of the website after its first-time deployment. It takes 1.92 seconds to load the design page on a laptop.

To make the page load faster, I made the images smaller -- on average, image size was reduced by 70%. I then republished the website, cleared all browser caches and made another performance measurement. The loading time was reduced to 1.15 seconds. The pictures displayed on other pages have also been optimised with smaller file sizes.

HTML Validation

The website has been validated using online validation tools. Errors were caused by a missing start tag and incorrect usage of the "for" label.

After making the corrections, the website was validated again and it passed the test.

Accessibility

The accessibility check was carried out using WAVE. The test results showed 9 low contrast errors in the header and footer. The cause of the errors was the font colour being too light.

To find a suitable font colour, Contrast Checker was used. The font colour was changed to black.

After updating the font colour to black, another accessibility test was carried out and passed.
